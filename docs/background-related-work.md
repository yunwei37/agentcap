## Novelty Verdict
- Overall same-claim risk: medium-high. I would not claim the broad idea is unquestionably novel. After the expanded 2026 corpus, the safe claim is narrower: IntentCap is a run-centric authorization compiler/checker that packages context influence over authority-bearing decisions as intent-carrying, proof-carrying, attenuable capability leases across agent extensions.
- Claims safe to keep: context influence as an explicit lease predicate; structured intent certificates as the root of run-time authority; provenance-authorized decision classes; deterministic checker outside the LLM TCB; cross-extension run-centric authorization across Skills, MCP tools, local scripts, and subagents.
- Claims to narrow or drop: "first permission model for Skills/MCP" is too broad and conflicts with Progent, SkillGuard, and SkillScope. "First work to connect user intent to agent execution" conflicts with Intent-to-Execution Integrity and contextual-security papers. "Solves prompt injection" must not be claimed. "Policy synthesis for agents" is too close to Progent/ActPlane unless tied specifically to intent certificates, influence modes, and context-control provenance.
- Closest novelty risks: CaMeL and Prompt Flow Integrity for control/data-flow capability security; AgentSecBench and Formalizing LLM Agent Security for contextual/noninterference-style framing; Progent for programmable least privilege and LLM-generated policies; SkillGuard/SkillScope for Skill permissions and context/action governance; ActPlane for agent policy enforcement below the tool layer; DRIFT/Task Shield for dynamic prompt-injection defenses with utility/security numbers.
- Larger claim opportunity: portable intent-provenance-aware policy compiler for agent extension ecosystems. The paper should be about deciding which authority is justified and which context may influence a decision, not about building a new sandbox, a new extension interface, or another prompt filter.
- Absorbable ideas to import: AgentDojo oracles, CaMeL's explicit control/data split, Prompt Flow Integrity's privilege-escalation cases, AgentSecBench-style projection/noninterference metrics, Task Shield's objective-serving check, Progent-style deterministic fallback comparisons, SkillGuard/SkillScope Skill taxonomies, MCPTox and MCP description-code inconsistency cases, tau2/tau3 domain policies, and ActPlane-style backend lowering.
- Mandatory baselines: vanilla, static allowlist/tool filter, Task Shield, CaMeL or PFI-style flow baseline, DRIFT/PromptArmor if feasible, Progent-style policy, Skill manifest/SkillGuard/SkillScope-style policy, MCP exact-tool/server allowlist, OS enforcement only/ActPlane-style backend, LLM-only lease generation, model-internal authorization contrast, and expert-written oracle leases.
- PDF corpus status: 42 primary PDFs now live under `docs/reference/` and were verified locally as PDFs. This is a much stronger source corpus for the current novelty audit, but it is not a proof of literature completeness; a top-conference submission still needs full-text reading notes and reproduction decisions for the closest baselines: CaMeL, Prompt Flow Integrity, AgentSecBench/Formalizing LLM Agent Security, Progent, SkillGuard/SkillScope, and Task Shield/DRIFT.
- Next action: harden benchmark evidence with stronger exact-argument synthesis and non-oracle task-loop recovery after R079/R080, and turn R081's saved-result authority matrix into a blinded expert-oracle or independent validity-label study. Do not sync new eval datasets by default; downloading paper PDFs is allowed and has been done, but benchmark code/data imports still require explicit experiment scope.
