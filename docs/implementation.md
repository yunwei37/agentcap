## Integration Constraints
- Do not mutate the frozen workshop paper unless explicitly requested.
- Keep durable research memory in the canonical docs; raw benchmark outputs should go under `results/`.
- Record every nontrivial experiment command, commit, machine, seed/repetition policy, and result path in `docs/evaluation.md`.
- Avoid requiring model API keys for smoke tests. Use local/static traces first where possible.
- Treat benchmark setup failures as useful evidence and record them.
- Do not clone, sync, or bulk-download new eval datasets by default. Use web search and official metadata first, then ask before bringing a new dataset into `benchmarks/` or `results/`.
- Treat R075 broad/runtime gateway allows as risk evidence. R076 strict lowering shows that denying those policies removes broad/runtime allows but leaves only 11/61 reference actions exact-constrained in saved gateway replay. R077/R078 show that runtime placeholders improve diagnostics but do not improve exact coverage or gateway utility by themselves. R079 proves a strict non-oracle compiler-corpus task loop can execute as a safety lower bound. R091 proves that runtime placeholders can be bound safely to executed tool-result evidence and recover additional exact executions. R095 proves that repeated compiler corpora can be unioned without changing checker authority, but only improves bound references by one over R091. R099 proves active compiler-lease hints/fallback can recover two more bound references without broad authority. R104 shows runtime-evidence hints alone can be ignored by the model and even lose coverage relative to R099. R109 shows runtime-evidence hint-choice can increase exact bound-reference calls to 20 without gateway blocks or tool errors, but it also introduces 6 off-reference/wrong-candidate calls and still leaves 38 DB-feasible references missing. R114 shows high-impact value-proof/proof-probe semantics can block an insufficiently proven write and force read-before-write evidence collection, but still leaves 37 DB-feasible references missing and 0/11 official tool-oracle pass. The next repair target is intent-grounded candidate ranking, proof-completeness scoring, exact argument extraction, tool exposure/activation, and task planning rather than broadening static authority or blindly executing every visible candidate.
