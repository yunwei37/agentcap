Best-case theorem targets, if the system model is kept honest:

- No unauthorized influence: in any accepted trace, protected decisions have control provenance only from context labels authorized for that decision class.
- Intent-bounded authority: every high-impact event has an active lease derivable from trusted user intent, explicit object/sink selection, policy, or fresh approval.
- Delegation monotonicity: delegated leases are no broader than parent leases unless a trusted issuer creates a fresh lease.
- Least-privilege synthesis, scoped: for a given effect plan and capability catalog, generated leases have lower risk than static ACLs and are closer to expert oracle; do not claim global minimality.

Non-claims for the top-conference version:

- Do not claim prompt injection is solved.
- Do not claim the current compiler is complete or correct.
- Do not claim benchmark-scale online utility preservation until TC4 has a real non-oracle task-loop result.
- Do not claim approval-burden reduction before an approval simulator or user study exists.
- Do not claim OS/sandbox enforcement is unnecessary; IntentCap decides policy provenance and can lower to those backends.

## Baselines And Fairness Policy
- Same model, temperature, tool descriptions, task order, and prompt budget wherever possible.
- Separate policy power from model power: compare wrappers around the same underlying agent.
- Do not compare a fully implemented IntentCap runtime against a paper-only baseline without marking reproduction status.
- For AuthGraph/PACT/AIRGuard/FIDES/RTBAS/CaMeL/PFI/NeuroTaint/Progent/PCAS/AgentSpec/AgentGuard/AgentBound/Task Shield/DRIFT/SkillGuard/SkillScope, prefer artifact reproduction. If unavailable, report as conceptual or trace-level partial baseline, not a numeric win.
- AgentSecBench/Formalizing LLM Agent Security/Intent-to-Execution Integrity should be used to audit theorem wording and baseline taxonomy even if they are not directly runnable baselines.
- Expert oracle leases should be written before seeing IntentCap output for each sampled task.
- Dataset acquisition policy after 2026-07-03: do not clone, sync, or bulk-download new eval datasets by default. Use official web pages, papers, data cards, and repository metadata to choose candidates first; run only existing local artifacts or explicitly approved small downloads.

## Workloads, Metrics, Oracles, And Run Order
### Workloads
- Primary security: AgentDojo because it has stateful environments, formal utility/security checks, multiple tools, and realistic injection endpoints.
- Breadth security: InjecAgent because it spans many user and attacker tools.
- MCP security: MCPTox or a compatible local MCP poisoning subset.
- Benign utility and policy pressure: tau2/tau3 and MCP-Bench/LiveMCPBench.
- Additional web-survey candidates, not synced locally: Skill-Inject and MCPSecBench for high-priority Skill/MCP security expansion, ToolSandbox and WorkBench for stateful utility/recovery, TheAgentCompany for multi-app authority, AgentHarm/HarmfulSkillBench/HarmActionsEval/Agent-SafetyBench for safety-protocol-gated denial stress tests, WebArena/WorkArena/BrowserGym for browser/enterprise workflows, OSWorld for desktop/file authority, GAIA and BrowseComp for long-horizon web-context provenance, and SWE-bench for coding-agent repo authority.
- Custom mixed workflow: PDF extraction -> local script -> GitHub issue, used for explanatory case study and unit tests.

### Metrics
- Attack success rate.
- Wrong-sink action rate.
- Data exfiltration rate.
- Unauthorized approval-scope inflation.
- Unauthorized delegation accepted.
- High-impact decisions with untrusted control provenance.
- Benign task completion.
- False denial and recovery rate.
- Risk-weighted authority score.
- Exposed tool/MCP method/path/network/sink/influence breadth.
- Checker invalid proposal rejection and valid proposal acceptance.
- Latency/overhead for online enforcement when implemented.

### Oracles
- Benchmark-provided state checks where available.
- IntentCap protected-decision oracle: denied if a protected decision has control provenance from a context source lacking the required influence mode.
- Expert lease oracle for authority minimization.
- Manual review protocol for ambiguous natural-language summaries, recorded as non-final evidence.

### Run Order
1. Sanity: local PDF wrong-sink trace with hand-written labels and leases.
2. Setup probe: AgentDojo install/import and one documented benchmark command.
3. Probe: AgentDojo subset with offline checker over hand-labeled traces.
4. Baseline: vanilla/static allowlist on same subset.
5. Expansion probe: InjecAgent or MCPTox adapter.
6. Utility probe: tau2/tau3 or MCP-Bench benign tasks.
7. Main runs: broaden task count and baselines after adapters stabilize.
